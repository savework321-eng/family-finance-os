# Coding Standards

## General Principles

- Clean Architecture
- SOLID Principles
- DRY
- KISS
- Separation of Concerns

---

# File Rules

Maximum 500 lines per file.

One responsibility per file.

No duplicated code.

---

# Naming Convention

Classes

PascalCase

Variables

camelCase

Constants

UPPER_SNAKE_CASE

Files

PascalCase for Classes

camelCase for Utilities

---

# Documentation

Every function must use JSDoc.

Every module must contain documentation.

---

# Configuration

No magic strings.

No hardcoded values.

All constants must be stored in Core_Config.

---

# Error Handling

Never ignore exceptions.

Always return standardized responses.

---

# Logging

Every critical action must be logged.

Every error must be logged.

---

# Source Control

Meaningful commit messages.

One logical change per commit.

No direct changes on production branch.

---

# Code Review

Every completed module must be reviewed before merge.
